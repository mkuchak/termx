termx turns your phone into a control surface for Claude Code running on any SSH-reachable server. It opens your tmux sessions 1:1 so the same terminals you started from your laptop show up on the phone, with Claude-running sessions auto-detected and badged. The optional companion binary (termxd) writes an event stream to ~/.termx/ on the server, giving the phone push-style notifications for build failures, approval prompts, and finished agent tasks. Permission prompts from Claude Code's PreToolUse hook surface as native dialogs you can approve from the lock screen — no parallel whitelist to maintain.

termx is fully free and open source and ships with zero telemetry. SSH private keys and stored passwords never leave the phone in plaintext — they sit in the Android Keystore and unlock on biometric. The push-to-talk feature uses your own Gemini API key (you paste it in Settings; it is stored in the vault, never sent to any termx server). There is no cloud account, no analytics, no paid tier, and no server-side component that someone else operates. Every feature that would otherwise require a relay (FCM, config sync) is either self-hostable or a file-based tail over your existing SSH connection. If you like the project, donations are welcome, but nothing is gated.
